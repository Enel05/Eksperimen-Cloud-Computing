Pengantar

Apa itu Cloud Computing?

Cloud computing atau komputasi awan adalah model layanan komputasi yang memungkinkan pengguna memakai sumber daya seperti server, penyimpanan, jaringan, aplikasi, dan daya pemrosesan melalui internet. Dengan cloud, layanan tidak harus dijalankan pada komputer lokal, tetapi dapat diakses dari mana saja selama tersedia koneksi jaringan.


Konsep cloud computing berkembang dari komputasi terdistribusi dan grid computing. Penyedia layanan cloud mengelola infrastruktur utama, sedangkan pengguna dapat memakai layanan sesuai kebutuhan tanpa harus membeli, memasang, dan merawat perangkat keras sendiri.

Cloud computing mengubah cara organisasi memakai teknologi: dari membeli infrastruktur sendiri menjadi menggunakan layanan digital yang fleksibel, elastis, dan berbasis pemakaian.

Karakteristik

Karakteristik Utama Cloud Computing

1

Resource Pooling

Sumber daya seperti CPU, memori, storage, dan bandwidth dikumpulkan oleh penyedia layanan untuk melayani banyak pengguna secara efisien.

2

Broad Network Access

Layanan cloud dapat diakses melalui jaringan dari berbagai perangkat, seperti laptop, smartphone, tablet, dan workstation.

3

Measured Service

Penggunaan resource dapat dipantau, diukur, dan dilaporkan berdasarkan parameter seperti storage, bandwidth, processing, dan jumlah pengguna aktif.

4

Rapid Elasticity

Kapasitas layanan dapat dinaikkan atau diturunkan dengan cepat sesuai kebutuhan pengguna atau beban kerja.

5

Self Service

Pengguna dapat mengatur layanan secara mandiri, seperti menambah kapasitas atau membuat server virtual tanpa interaksi langsung dengan penyedia layanan.

6

Pay as You Go

Biaya cloud mengikuti penggunaan. Organisasi membayar resource yang dipakai, bukan membeli seluruh infrastruktur dari awal.

Jenis Layanan

SaaS, PaaS, dan IaaS

S

Software as a Service

SaaS adalah layanan cloud yang menyediakan software siap pakai. Contohnya Gmail, Google Docs, Office 365, Adobe Creative Cloud, dan aplikasi bisnis online. Pengguna tidak perlu mengelola server atau instalasi aplikasi secara manual.

P

Platform as a Service

PaaS menyediakan platform untuk menjalankan dan mengembangkan aplikasi. Pengembang dapat fokus membangun aplikasi tanpa harus mengurus server, sistem operasi, database, atau runtime secara langsung.

I

Infrastructure as a Service

IaaS menyediakan infrastruktur virtual seperti CPU, RAM, storage, dan jaringan. Pengguna dapat menyewa virtual machine dan mengatur kapasitas sesuai kebutuhan.

Deployment Model

Model Penerapan Cloud

Public Cloud

Layanan cloud tersedia untuk umum dan dikelola oleh penyedia cloud. Contohnya layanan seperti Gmail, Office 365, Google Apps, dan berbagai layanan cloud publik lainnya.

Private Cloud

Layanan cloud digunakan untuk kebutuhan internal organisasi. Model ini memberikan kontrol lebih besar terhadap keamanan, konfigurasi, dan data.

Hybrid Cloud

Hybrid cloud menggabungkan public cloud dan private cloud. Model ini digunakan untuk menyeimbangkan fleksibilitas, kontrol, keamanan, dan efisiensi biaya.

Community Cloud

Community cloud dibangun untuk komunitas tertentu yang memiliki kebutuhan, standar, kebijakan, atau kepentingan yang sama.

Manfaat

Kelebihan Cloud Computing

  1. Skalabilitas Kapasitas server, storage, dan layanan dapat ditambah atau dikurangi sesuai kebutuhan.
  2. Efisiensi Biaya Cloud menggunakan model pay-as-you-go sehingga mengurangi investasi awal perangkat keras.
  3. Aksesibilitas Layanan dapat diakses dari berbagai lokasi selama pengguna terhubung ke internet.
  4. Pemulihan Data Penyedia cloud umumnya menyediakan backup, enkripsi, dan sistem keamanan tambahan.

Tantangan

Tantangan Cloud Computing

Cloud computing juga memiliki tantangan. Beberapa di antaranya adalah ketergantungan pada koneksi internet, risiko keamanan dan privasi data, keterbatasan kontrol terhadap infrastruktur, serta kebutuhan integrasi antara aplikasi lama dan sistem cloud.


Karena itu, penggunaan cloud perlu dirancang dengan strategi yang matang, termasuk pengelolaan keamanan, pemantauan biaya, dan tata kelola operasional.

Perkembangan Teknologi

Tren Cloud Computing

H

Hybrid dan Multi-Cloud

Organisasi semakin sering memakai kombinasi public cloud, private cloud, dan beberapa penyedia cloud sekaligus agar lebih fleksibel dan tidak bergantung pada satu vendor.

AI

AI dan Machine Learning

Cloud mendukung layanan AI dan machine learning untuk otomasi, analisis data, prediksi, rekomendasi, dan pengembangan layanan digital yang lebih cerdas.

CN

Cloud Native

Teknologi seperti Docker, Kubernetes, dan containerization membantu deployment, scaling, dan pengelolaan aplikasi modern secara lebih cepat dan konsisten.

Financial Operations

Introduction to FinOps

FinOps adalah praktik budaya dan kerangka kerja operasional yang menyatukan tim teknologi, keuangan, dan bisnis. Tujuannya bukan sekadar mengurangi biaya cloud, tetapi memaksimalkan nilai bisnis dari setiap biaya cloud yang dikeluarkan.


Pada model tradisional, perusahaan memakai CapEx atau investasi besar di awal untuk membeli perangkat keras. Pada model cloud, biaya berubah menjadi OpEx berbasis pemakaian. Model ini fleksibel, tetapi juga dapat menjadi sulit diprediksi tanpa pengelolaan biaya yang baik.

FinOps membantu organisasi menjawab pertanyaan penting: “Apakah biaya cloud yang kita keluarkan sudah sebanding dengan nilai bisnis yang dihasilkan?”

Framework FinOps

Inform, Optimize, Operate

I

Inform

Tahap ini berfokus pada visibilitas biaya. Organisasi perlu memahami biaya cloud, melakukan tagging, alokasi biaya, dashboard, dan pelaporan yang jelas.

O

Optimize

Tahap ini mencari peluang penghematan seperti rightsizing, reserved instances, spot instances, menghapus resource idle, dan memilih layanan yang lebih efisien.

R

Operate

Tahap ini mengintegrasikan efisiensi biaya ke operasional harian melalui governance, KPI, metrics, policy enforcement, dan kolaborasi antar tim.

Stakeholder

Pemangku Kepentingan dalam FinOps

Persona Tanggung Jawab Fokus FinOps
Engineering Membangun, menjalankan, dan mengelola infrastruktur serta aplikasi cloud. Efisiensi penggunaan resource, rightsizing, dan pengurangan resource idle.
Finance Mengelola anggaran, pelaporan biaya, dan perencanaan keuangan. Prediksi biaya, kontrol anggaran, chargeback, dan showback.
Product Owner Menentukan strategi produk dan memastikan fitur memberikan nilai bagi pengguna. Analisis biaya per fitur, produk, pelanggan, atau transaksi bisnis.
Executive Mengambil keputusan strategis berdasarkan biaya, performa, dan pertumbuhan bisnis. ROI, efisiensi organisasi, dan nilai bisnis dari investasi cloud.

Unit Economics

Mengukur Nilai Bisnis Cloud

FinOps tidak hanya menanyakan “berapa total biaya cloud?”, tetapi juga “berapa biaya untuk menghasilkan satu transaksi bisnis?”. Pendekatan ini disebut unit economics.


Contohnya, organisasi dapat menghitung biaya cloud per pengguna, per transaksi, per pesanan, per fitur, atau per layanan. Dengan begitu, biaya cloud dapat dihubungkan langsung dengan nilai bisnis.

Contoh Pertanyaan Unit Economics

  1. Biaya per pelanggan Berapa biaya cloud untuk melayani satu pelanggan aktif?
  2. Biaya per transaksi Berapa biaya cloud untuk memproses satu transaksi?
  3. Biaya per fitur Fitur mana yang paling mahal dan apakah nilainya sebanding?

Maturity Model

Crawl - Walk - Run

1

Crawl

Organisasi mulai membuat visibilitas dasar biaya, tagging, dan laporan sederhana. Proses masih banyak dilakukan secara manual.

2

Walk

Kebijakan optimasi mulai dilakukan secara rutin. Kolaborasi antara engineering, finance, dan bisnis mulai terbentuk.

3

Run

Otomasi berjalan matang. Biaya cloud dipantau sebagai nilai strategis bisnis secara real-time dan menjadi bagian dari keputusan operasional.

Kesimpulan

Cloud Memberi Fleksibilitas, FinOps Memberi Kendali

Cloud computing memberikan kemudahan dalam mengakses resource komputasi secara fleksibel, cepat, dan skalabel. Namun, fleksibilitas ini juga membuat biaya cloud dapat berubah dengan cepat jika tidak dikelola secara benar.

  • Cloud computing membantu organisasi mengakses teknologi tanpa harus membeli infrastruktur sendiri.
  • SaaS, PaaS, dan IaaS memberikan pilihan layanan sesuai kebutuhan pengguna.
  • Public, private, hybrid, dan community cloud digunakan sesuai kebutuhan keamanan, kontrol, dan fleksibilitas.
  • FinOps menyatukan teknologi, keuangan, dan bisnis untuk mengelola biaya cloud secara bertanggung jawab.
  • Tujuan akhir FinOps bukan hanya penghematan, tetapi memaksimalkan nilai bisnis dari penggunaan cloud.
